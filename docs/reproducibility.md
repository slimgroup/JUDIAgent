# Reproducibility Guide

This guide records the public-release path for reproducing the JUDIAgent
software environment, smoke tests, benchmark prompts, and publication-facing
figures.

## 1. Clone And Install Python Dependencies

```bash
git clone https://github.com/haoyunl2/JUDIAgent.git
cd JUDIAgent
uv venv
source .venv/bin/activate
uv sync
```

`uv.lock` is committed and should be used for reproducible Python dependency
resolution.

## 2. Configure Julia

JUDIAgent expects Julia to run with this repository as the active project:

```bash
julia --project=. -e 'import Pkg; Pkg.instantiate()'
```

On shared clusters, run the first Julia/JUDI instantiate or precompile pass on
an interactive compute node. See `docs/devel-pace.md` for the PACE workflow.

## 3. Configure Credentials

Create a local `.env` file:

```bash
cp .env.example .env
```

Fill only the providers you use. The default remote setup uses a DeepSeek chat
model and OpenAI embeddings, so it needs `DEEPSEEK_API_KEY` and
`OPENAI_API_KEY`. `LANGSMITH_API_KEY` is optional unless tracing or GUI workflows
are enabled.

## 4. Verify The Codebase

Run the low-cost import and graph wiring checks first:

```bash
uv run pytest tests/integration_tests/test_entrypoints.py
```

For a fuller local validation pass:

```bash
uv run pytest
```

These tests avoid live LLM calls and heavy Julia simulations unless explicitly
requested.

## 5. Run The Agents

Standard iterative agent:

```bash
uv run python examples/agent.py
```

Autonomous ReAct agent:

```bash
uv run python examples/autonomous_agent.py
```

Generated scripts and artifacts are written to `scripts/` and `outputs/`; those
directories are ignored by git so runs do not overwrite publication source files.

## 6. First Retrieval Run

Retrieval indexes are intentionally not tracked in git:

- `src/judiagent/rag/retriever_store/`
- `src/judiagent/rag/loaded_store/`

The first call to a retrieval tool loads the JUDI corpus, chunks source files,
builds embeddings, and persists the vector index under those ignored cache
directories. With the default remote embedding model this requires
`OPENAI_API_KEY` and network access. Local embedding experiments can set
`LOCAL_MODELS = True` in `src/judiagent/settings.py` or override the LangGraph
runtime config to use `ollama:nomic-embed-text`.

The cache can be deleted and rebuilt when the corpus or embedding model changes.
Do not commit rebuilt index files unless you are intentionally publishing a
versioned retrieval snapshot.

## 7. Benchmark Prompt Catalog

The benchmark prompts live in `benchmarks/prompts.yaml`. Each task records:

- the natural-language prompt,
- required workflow ingredients,
- acceptance criteria,
- task-aware metric guidance.

Use the task identifiers, for example `basic_2d_forward`, `rtm_basic`, and
`fwi_basic`, when comparing agent runs or configuring domain validation.

The prompt catalog is loadable from Python:

```python
from judiagent.benchmarks import get_benchmark_task, load_benchmark_tasks

tasks = load_benchmark_tasks()
rtm_task = get_benchmark_task("rtm_basic")
```

This loader validates the catalog schema and supports smoke tests. It is not a
full batch-evaluation harness; live benchmark runs should record the selected
agent mode, model, retrieval settings, `benchmark_task_id`, generated script,
and output artifacts.

## 8. Publication Figures And Draft Text

Publication-facing notes and figures live under `code_summary/`.

Workflow SVGs are synchronized from `media/` with:

```bash
uv run python code_summary/figures/generate_figures.py
```

Case-study figures generated by agent runs should be regenerated from the
validated scripts and artifacts recorded with the experiment. Avoid editing
binary figures by hand without also keeping the script or metadata that produced
them.

## 9. Third-Party Material

The retrieval corpus includes JUDI.jl documentation and examples. See
`NOTICE` and `src/judiagent/rag/THIRD_PARTY.md` before creating a public archive
or DOI, especially if you plan to redistribute notebooks, SEG-Y/HDF5 data, or
field-data processing examples.
